如果天性追求第一性原理，追求通晓底层原理，不满足于只是便宜行事，那么 AI 简直是福音。没有 Codex 和 Claude Code，我大概永远也不可能完成以下的尝（最近一年多断断续续搞的）：
1、写个 toy linux kernel 出来，麻雀虽小五脏俱全地能多任务，能管内存，能 bootload，filesystem 能用。
2、写个玩具版本浏览器内核出来，虽然很捞完全没法和 Chrome 比但是基础 js 渲染，隔离，后台什么的功能都是齐全的。
3、把家庭 K8S homelab 集群搭建出来（因为有很多 GPU 节点和边缘节点）并且持久化维护。
4、按照 Ben Eater 的手搓 CPU 教程把 8bit 面包板 CPU 搭出来。
5、写个 toy PyTorch 出来，不用科学计算库纯 build from scratch
6、写个完整的能跑的小数据库出来（B+ Tree，多读多写，indexing）
7、写个能跑的 3D 物理引擎出来
8、写个微小 Claude Code 出来（强调 Harness engineer 的优化）
虽然都不太能直接投入生产吧，但是对底层理解的增益是实打实的，投入到科研和创业里自觉非常有帮助。
本科教育里写的那些零散的 malloc，shell，compiler，DB 模块，科学计算模块，现在回头看看觉得那些项目设计的真的不那么合理（CMU 和 MIT 的可能好一些吧），而且说实话一学期的时间完全只能学个皮毛，完全不扎实，A+ 了又怎样，还不是会感觉 imposter syndrome 觉得自己没学明白不配。
没有现代 Coding Agent 的话，光查文档去谋划个大概的骨架圈定个范围大概就要死活要活了。拿别人的仓库过来根本无法直接下手客制化，不是自己设计的很不容易搭建起 cognitive map，学习效果不佳（有种娃还是自己亲手养大的才能感觉通透的感觉）。自己从零写出来的就像自己家后花园，就很轻车熟路，而且每个模块往往都有多个不同的尝试版本尝试轨迹，非常方便做对比实验和一些 “大胆实验”（没错，就是要压力测试把电脑炸掉 手动滑稽）写完这些再去搞 Harness，优化后端分库分表，给爬虫加反 Captcha，搞 GPU 优化，搭嵌入式系统，感觉气儿都顺了，神清气爽。
